# Journal 3: Deploy leah.lan di Home Server

> Catatan: cara pasang LEAH di home server sendiri sampai bisa
> diakses via http://leah.lan — lengkap dengan dnsmasq, nginx, systemd.

---

## 1. Yang Terjadi

Setelah frontend dan backend selesai, aku ingin LEAH bisa diakses
dari browser teman-teman di rumah lewat `http://leah.lan` — bukan
`localhost:5173` atau `localhost:8080`.

Ini butuh 3 komponen:

```
Browser → DNS (dnsmasq) → Nginx (proxy) → Go binary
                    ↓
          leah.lan = 192.168.1.2
```

## 2. dnsmasq — Biar leah.lan dikenal di network

dnsmasq adalah DNS server local. Dia yang ngasih tahu jaringan di rumah:
"kalau ada yang minta `leah.lan`, kirim ke `192.168.1.2`".

File konfig: `/etc/dnsmasq.d/leah.lan.conf`
Isinya cuma satu baris:

```
address=/leah.lan/192.168.1.2
```

Setelah itu restart: `sudo systemctl restart dnsmasq`.

Ini standar untuk semua project di home server (fitness.lan, jyavani.lan, dll).

## 3. Nginx — Web Server + Reverse Proxy

Nginx tugasnya:
1. **Serve file statis** — landing page React yang sudah di-build ada di `frontend/dist/`
2. **Proxy API** — kalau ada request ke `/api/*`, teruskan ke Go binary di `localhost:8080`

Kenapa perlu Nginx? Bukannya Go sudah bisa jadi web server sendiri?

Iya, Go bisa. Tapi di production, Nginx menangani:
- **SSL/HTTPS** — kalau nanti pakai domain beneran
- **Serve static files** — lebih cepat dari Go untuk file HTML/CSS/JS
- **Security** — proteksi dari serangan umum
- **Caching** — file statis di-cache browser

Config nginx-nya simpel:

```nginx
server {
    listen 80;
    server_name leah.lan;

    root /var/www/leah.lan/frontend/dist;

    # API → proxy ke Go
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
    }

    # Sisanya → file statis (React SPA)
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

## 4. Systemd Service — Biar Auto-start & Auto-restart

Systemd adalah "manager proses" di Linux. Dia yang ngatur service.
Systemd service = cara profesional menjalankan aplikasi di Linux.

File: `/etc/systemd/system/leah.service`

```ini
[Unit]
Description=LEAH - List Everything Assets & Helpdesk
After=network.target postgresql.service

[Service]
Type=simple
User=adam
WorkingDirectory=/var/www/leah.lan/backend
ExecStart=/var/www/leah.lan/backend/leah
Restart=always
RestartSec=3
Environment=DATABASE_URL=postgres://leah:leah@localhost:5432/leah?sslmode=disable
Environment=PORT=8080

[Install]
WantedBy=multi-user.target
```

Yang penting di sini:
- **`After=postgresql.service`** — LEAH jalan setelah PostgreSQL siap
- **`Restart=always`** — kalau crash, restart otomatis dalam 3 detik
- **`Environment=`** — variabel lingkungan, gantinya `.env`

Cara pakenya:
```bash
sudo systemctl start leah     # mulai
sudo systemctl enable leah    # auto-start saat boot
sudo systemctl status leah    # cek status
sudo journalctl -u leah -f    # lihat log real-time
```

## 5. LEAH Development vs Production

| Mode | Backend | Frontend | Akses lewat |
|------|---------|----------|-------------|
| **Dev** | `go run ./cmd/api` (hot reload?) | `npm run dev` (Vite) | `localhost:5173` |
| **Prod** | `systemd → ./leah` (binary) | Nginx → `frontend/dist/` | `leah.lan` |

## 6. leah.sh — Control Script

Biar gak perlu hafal command systemd, kubuat `leah.sh` di root project:

```bash
./leah.sh build     # compile backend + frontend
./leah.sh start     # mulai service (systemd)
./leah.sh stop      # hentikan service
./leah.sh restart   # restart service
./leah.sh status    # cek status
./leah.sh logs      # lihat log real-time
./leah.sh dev       # petunjuk mode development
```

## 7. Perbandingan: PHP Native vs Go

```
Dulu (PHP):
  git pull                                        → selesai ✅
  (file .php langsung jalan, gaperlu compile)

Sekarang (Go):
  git pull
  cd backend && go build -o leah ./cmd/api         → compile (3-5 detik)
  cd frontend && npm run build                     → build frontend (2 detik)
  sudo systemctl restart leah                      → restart (1 detik)
                                                    ✅ selesai

Ada tambahan langkah "build" + "restart".
Tapi setelah jalan: memory 5MB, tanpa interpreter, tanpa dependency.
```

## 8. Catatan Penting

- **CORS origins** di systemd service masih `http://localhost:5173` — ini untuk dev mode.
  Di production (leah.lan), frontend dan backend satu domain via Nginx, jadi CORS gak diperlukan.
  Tapi kalau ada masalah, bisa diubah jadi `http://leah.lan` atau `*`.

- **Binary `backend/leah` sudah di `.gitignore`** — karena binary di-generate dari kode,
  gak perlu di-commit ke git. Setiap clone project, tinggal `go build`.

- **Kalau pindah server** — cukup copy folder project (tanpa node_modules, tanpa binary),
  install PostgreSQL, jalankan `001_init.sql`, build ulang, dan start service.
