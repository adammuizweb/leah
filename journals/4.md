# Journal 4: Cloudflare Tunnel — Akses LEAH dari Internet

> Cara `leah.jyavani.com` bisa diakses dari luar rumah
> pakai Cloudflare Tunnel — tanpa port forwarding, tanpa IP publik.

---

## 1. Masalah

LEAH hanya jalan di network rumah (`leah.lan` via dnsmasq).
Kalau di luar rumah (cafe, kantor, pakai data HP), gak bisa akses.

Solusi传统的:
- **Port forwarding** — buka port 80/443 di router → **berbahaya, IP terekspos**
- **Dynamic DNS** — repot, tetap buka port

Solusi modern:
- **Cloudflare Tunnel** — tunnel keluar dari server ke Cloudflare, inbound lewat Cloudflare

## 2. Cara Kerja Cloudflare Tunnel

```
User → https://leah.jyavani.com
          ↓ DNS Cloudflare
          ↓ CNAME → 4cfcf...cfargotunnel.com
          ↓
    Cloudflare Edge Network
          ↓ (koneksi keluar dari server ke Cloudflare)
    cloudflared (di home server)
          ↓
    localhost:80 (Nginx)
          ↓
    localhost:8080 (Go binary)
```

Yang penting:
- **Koneksi KELUAR** dari server ke Cloudflare — bukan masuk
- Server gak perlu port terbuka sama sekali
- SSL/HTTPS otomatis dari Cloudflare
- IP server tetap private

## 3. Yang Dilakukan

Ada 2 tunnel di home server:

| Tunnel ID | Akun Cloudflare | Domain |
|-----------|----------------|--------|
| `4f218d22...` | adammuiz.com | adammuiz.com, ai.adammuiz.com |
| `4cfcf86c...` | jyavani.com | jyavani.com, cdn.jyavani.com, **leah.jyavani.com** |

### Langkah:

**1. Tambah ingress rule di cloudflared config**

File: `/etc/cloudflared/jyavani.yml`

```yaml
ingress:
  - hostname: jyavani.com
    service: http://localhost:80
  - hostname: www.jyavani.com
    service: http://localhost:80
  - hostname: cdn.jyavani.com
    service: http://localhost:80
  - hostname: leah.jyavani.com    # ← tambah ini
    service: http://localhost:80
  - service: http_status:404       # catch-all HARUS di paling bawah
```

Urutan penting: specific hostname di atas, catch-all `http_status:404` di paling bawah.

**2. Restart tunnel**

```bash
sudo systemctl restart cloudflared-jyavani
```

**3. DNS record di Cloudflare Dashboard**

Bukan A record, tapi **CNAME** yang指向 tunnel:

```
Type:   CNAME
Name:   leah
Target: 4cfcf86c-e8bb-47e4-a727-e38e8d5f66b2.cfargotunnel.com
Proxy:  Proxied (orange cloud) ☁️
```

Ini yang bikin bingung: DNS record-nya bukan指向 IP rumah, tapi指向 tunnel ID Cloudflare.
Cloudflare yang urus routing-nya ke server via tunnel.

## 4. Kenapa Gak Bisa via CLI?

Ada 2 akun Cloudflare berbeda:
- `adammuiz.com` — cert.pem ada di `~/.cloudflared/`
- `jyavani.com` — cert.pem ada di `/root/.cloudflared/` (akses terbatas)

Karena `leah.jyavani.com` domain jyavani.com, DNS record-nya cuma bisa diatur
lewat dashboard jyavani.com — bukan via CLI.

## 5. Hasil Akhir

```
https://leah.jyavani.com   ✅ bisa diakses dari mana pun
  ├── Landing page         ✅
  ├── /dashboard           ✅ (app shell)
  ├── /api/health          ✅ (backend via tunnel → nginx → go)
  └── SSL/HTTPS            ✅ otomatis dari Cloudflare
```

## 6. Catatan Penting

- **Nginx tetap jalan** — cloudflared tunnel指向 `localhost:80` (Nginx),
  bukan langsung ke Go binary. Nginx yang menentukan routing (static files vs API proxy).
- **Cloudflare SSL** — HTTPS diurus Cloudflare. Server cukup HTTP di port 80.
- **Kalau tunnel mati** — `sudo systemctl restart cloudflared-jyavani`
- **Gak perlu port forwarding** — ini yang paling penting. Router tetap aman.
