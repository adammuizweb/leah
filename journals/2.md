# Journal 2: Frontend Jalan Sendiri, Backend Jalan Sendiri

> Ternyata setelah nyobain `npm run dev`, aku baru paham —
> frontend dan backend itu proses terpisah.

---

## 1. Yang Baru Aku Paham

Frontend React (Vite) jalan di `http://localhost:5173/` — itu process sendiri.
Backend Go jalan di `http://localhost:8080/` — itu process sendiri juga.

**Mereka gak perlu satu folder, gak perlu satu proses, gak perlu satu bahasa.**

Ini persis kayak project AI-ku dulu:
```
ollama (backend API)  →  localhost:11434
web UI Python/JS      →  localhost:3000
```

Sama persis polanya:
```
LEAH backend (Go)     →  localhost:8080  →  ngasih JSON (data)
LEAH frontend (React) →  localhost:5173  →  ngasih HTML/CSS/JS (tampilan)
```

Bedanya cuma: kalau ollama backend-nya API AI, kalau LEAH backend-nya API database.

## 2. Cara Mereka "Ngobrol"

Frontend gak bisa langsung akses database PostgreSQL. Dia harus lewat backend.

```
Browser → localhost:5173 (Vite)
               ↓ proxy (vite.config.ts)
          localhost:8080 (Go API)
               ↓
          PostgreSQL (database)
```

Vite punya fitur **proxy**. Waktu frontend minta `http://localhost:5173/api/tickets`,
Vite otomatis terusin ke `http://localhost:8080/api/tickets`.

Ini cuma untuk development. Di production, yang jadi proxy adalah Nginx.

## 3. Cara Jalanin Keduanya

Sekarang butuh **2 terminal** (atau 1 terminal + background):

```bash
# Terminal 1 — Backend (Go)
cd /var/www/leah.lan/backend
DATABASE_URL="postgres://leah:leah@localhost:5432/leah?sslmode=disable" ./leah

# Terminal 2 — Frontend (React)
cd /var/www/leah.lan/frontend
npm run dev
```

Atau pake Makefile yang sudah kubuat:
```bash
make dev-backend   # jalanin Go
make dev-frontend  # jalanin React (di terminal lain)
```

## 4. Di Production Nanti

Gak perlu dua terminal. Nginx jadi jembatannya:

```
User → Nginx (:80)
          │
          ├── / → file statis dari frontend/dist/
          │
          └── /api/* → proxy ke localhost:8080 (Go binary)
```

Nginx serve frontend (file HTML/CSS/JS hasil build) dan proxy API request ke Go binary.
Cukup 1 domain, 1 port (80), tapi tetap ada 2 proses di belakang.

## 5. Kenapa Dipisah?

Dulu di PHP native (CMS), frontend dan backend campur jadi satu:
```
PHP-file.php → render HTML + logic sekaligus
```

Dengan React + API:
- Backend urus **data** (JSON) — bisa dipake oleh aplikasi lain (mobile, third-party)
- Frontend urus **tampilan** — bisa diganti tanpa sentuh backend

Pemisahan ini disebut **decoupling**. Sama seperti project AI dulu: kita bisa ganti
web UI (pake yang berbeda) tanpa perlu ganti ollama-nya.
