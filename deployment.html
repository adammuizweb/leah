🚀 LEAH — Deployment Guide

Dari kode di laptop sampai jalan di production

Prasyarat: Dokumen ini asumsinya kamu sudah familiar dengan PHP native deployment (copy file → setting nginx → done). Kita akan bandingkan dengan Go supaya kamu paham perbedaannya.

1. Filosofi: PHP vs Go

🐘 PHP Native (Jyavani CMS)

Interpreter — setiap request, PHP-FPM baca file .php, interpretasi, jalanin.

Runtime di server: PHP-FPM harus terinstall, file .php harus ada di disk.

Update: ganti file .php di server, langsung kebaca.

Dependency: 0 (kamu pure native).

🐹 Go (LEAH)

Compiled — kode Go di-compile jadi satu file binary (file executable).

Runtime di server: Nol. Gak perlu install Go di server. Cuma perlu 1 file binary.

Update: compile ulang → matikan binary lama → jalankan binary baru.

Dependency: Semua library Go sudah "tertanam" di dalam binary. Satu file = seluruh aplikasi.

Ilustrasi Perbedaan

PHP Native:
  Server:
    - Install PHP 8.4 + FPM
    - Install MariaDB
    - Copy semua file .php ke /var/www/leah/
    - Konfigurasi Nginx → proxy ke PHP-FPM
    - ✅ Selesai

  Setiap ada request:
    Nginx → PHP-FPM → baca file .php → jalanin → balikin HTML

Go:
  Server:
    - Install Go (CUMA UNTUK BUILD — bisa di laptop kamu sendiri)
    - Build → hasilnya 1 file: "leah"
    - Copy file "leah" ke server
    - Jalankan "./leah" (langsung jadi web server sendiri)
    - Konfigurasi Nginx → proxy ke "leah" (di port 8080)
    - ✅ Selesai

  Setiap ada request:
    Nginx → "leah" (binary yang sudah jalan) → balikin JSON

Intinya: PHP butuh interpreter di server. Go gak butuh apa-apa — binary-nya sudah mengandung semua yang dibutuhkan (termasuk web server).

2. Dua Cara Deploy LEAH

Ada 2 cara, kamu pilih sesuai selera:

Cara A: Binary Langsung (Go murni, tanpa Docker)

Ini yang paling simpel. Cocok buat kamu.

  +---------------------------+
  | LAPTOP / DEV              |
  |                           |
  |   go build -o leah ./cmd  |
  |       ↓                   |
  |   Hasil: file "leah"      |
  |   (ukuran ~20MB)          |
  +---------------------------+
            ↓ scp / rsync
  +---------------------------+
  | VM PRODUCTION             |
  |                           |
  |   ./leah &                |
  |   (jalan sebagai proses)  |
  |                           |
  |   Nginx → reverse proxy   |
  |   ke localhost:8080       |
  +---------------------------+

Cara B: Docker (Container)

Ini yang docker-compose.yml yang aku buat sebelumnya. Semua service (backend, frontend, database) jalan di container.

  +---------------------------+
  | VM PRODUCTION             |
  |                           |
  |   docker compose up -d    |
  |       ↓                   |
  |   Container: postgres     |
  |   Container: backend      |
  |   Container: frontend     |
  |                           |
  |   Nginx → reverse proxy   |
  |   ke container frontend   |
  +---------------------------+

Untuk kamu, aku saranin Cara A (binary langsung). Kenapa?

  • Gak perlu install Docker
  • Satu file binary — gampang di-copy, gampang di backup
  • Minimal dependency — sesuai filosofimu
  • Lebih gampang di-pahami karena transparan

3. Setup Fresh VM — Panduan Langkah demi Langkah

Anggap kamu punya VM baru (Debian/Ubuntu). Berikut step-by-step untuk deploy LEAH.

3.1 Install Prasyarat

# ===== 1. Update system =====
sudo apt update && sudo apt upgrade -y

# ===== 2. Install Nginx (web server) =====
sudo apt install nginx -y

# ===== 3. Install PostgreSQL =====
sudo apt install postgresql postgresql-contrib -y

# ===== 4. Install Go (CUMA UNTUK BUILD) =====
# Download & install Go
wget https://go.dev/dl/go1.22.linux-amd64.tar.gz
sudo tar -C /usr/local -xzf go1.22.linux-amd64.tar.gz
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.profile
source ~/.profile
go version  # → should show go1.22

Catatan penting: Go hanya untuk build. Setelah binary jadi, Go bisa di-uninstall. Server production gak butuh Go.

3.2 Build Binary

# ===== Di server production (atau di laptop, lalu di-copy) =====

# Clone kode
git clone https://github.com/adammuiz/leah.git /var/www/leah.lan
cd /var/www/leah.lan/backend

# Download dependency Go (hanya sekali)
go mod download

# Build binary
# Hasilnya: satu file bernama "leah" di folder yang sama
go build -o leah ./cmd/api

# Cek ukuran — sekitar 20MB
ls -lh leah
# -rwxr-xr-x 1 adam adam 20M Jun 30 10:00 leah

3.3 Setup Database

# ===== Setup PostgreSQL =====
sudo -u postgres psql

  -- Buat user database
  CREATE USER leah WITH PASSWORD 'password-kuat-di-sini';

  -- Buat database
  CREATE DATABASE leah OWNER leah;

  -- Keluar
  \q

# ===== Import skema database =====
sudo -u postgres psql -d leah -f /var/www/leah.lan/backend/migrations/001_init.sql

3.4 Siapkan Systemd Service (biar auto-restart)

Systemd memastikan binary LEAH jalan otomatis saat VM restart, dan restart otomatis kalau crash.

# ===== Buat file service =====
sudo nano /etc/systemd/system/leah.service

Isinya:

[Unit]
Description=LEAH - List Everything Assets & Helpdesk
After=network.target postgresql.service

[Service]
Type=simple
User=adam
WorkingDirectory=/var/www/leah.lan/backend
ExecStart=/var/www/leah.lan/backend/leah
Restart=always
RestartSec=5
Environment=DATABASE_URL=postgres://leah:password-kuat-di-sini@localhost:5432/leah?sslmode=disable
Environment=JWT_SECRET=rahasia-jwt-yang-kuat
Environment=PORT=8080

[Install]
WantedBy=multi-user.target
  
# ===== Aktifkan service =====
sudo systemctl daemon-reload
sudo systemctl enable leah
sudo systemctl start leah

# ===== Cek status =====
sudo systemctl status leah
# → "active (running)" ✅

# ===== Cek log =====
sudo journalctl -u leah -f

3.5 Setup Nginx Reverse Proxy

# ===== Buat konfigurasi Nginx =====
sudo nano /etc/nginx/sites-available/leah.lan

Isinya:

server {
    listen 80;
    server_name leah.lan;  # Ganti dengan domain kamu

    # ===== Frontend (React) — file statis =====
    root /var/www/leah.lan/frontend/dist;
    index index.html;

    # ===== API backend — proxy ke Go =====
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # ===== Fallback: semua request → index.html (buat React routing) =====
    location / {
        try_files $uri $uri/ /index.html;
    }
}
  
# ===== Aktifkan site =====
sudo ln -s /etc/nginx/sites-available/leah.lan /etc/nginx/sites-enabled/
sudo nginx -t       # test config
sudo systemctl reload nginx

3.6 Build Frontend untuk Production

React perlu di-build jadi file statis (HTML, CSS, JS) — bukan dijalankan sebagai dev server.

# ===== Install Node.js (cuma untuk build) =====
# Sama seperti Go — Node.js hanya perlu saat build, setelah itu bisa dihapus
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# ===== Build frontend =====
cd /var/www/leah.lan/frontend
npm install
npm run build

# Hasilnya ada di folder: frontend/dist/
# Ini file statis — bisa di-serve langsung oleh Nginx

# Cek hasil
ls dist/
# index.html  assets/

Setelah build selesai, Node.js bisa di-uninstall — server production hanya butuh file statis di frontend/dist/.

4. Cara Update LEAH

Salah satu kenyamanan PHP native: tinggal replace file .php, selesai. Di Go, ada beberapa langkah ekstra tapi masih simpel:

# ===== Setiap kali mau update LEAH ke versi baru =====

# 1. Pull kode terbaru
cd /var/www/leah.lan
git pull origin main

# 2. Build ulang binary Go
cd /var/www/leah.lan/backend
go build -o leah ./cmd/api

# 3. Build ulang frontend (kalau ada perubahan)
cd /var/www/leah.lan/frontend
npm install
npm run build

# 4. Restart service
sudo systemctl restart leah

# 5. (Opsional) Kalau ada migrasi database baru
sudo -u postgres psql -d leah -f /var/www/leah.lan/backend/migrations/002_nama_file.sql

# ✅ Selesai — total ~30 detik

Perbandingan: Update PHP vs Go

PHP Native:
  1. git pull
  2. ✅ Selesai (file .php langsung jalan)

Go:
  1. git pull
  2. go build (3-5 detik)
  3. systemctl restart leah (< 1 detik)
  4. ✅ Selesai

Perbedaan: Go perlu langkah "build" + "restart".
Tapi kompensasinya: performa jauh lebih cepat, memory lebih hemat.

5. Perbandingan: PHP Native vs Go LEAH

Kebutuhan 🐘 PHP Native 🐹 Go LEAH
Runtime di server PHP 8.4 + PHP-FPM ❌ Tidak perlu (binary sudah mandiri)
Web server Nginx (wajib) Nginx (opsional — sebagai reverse proxy / SSL)
Database MariaDB / MySQL PostgreSQL
File di server Ratusan file .php + .css + .js 1 binary + folder dist/ (static files)
Dev tools di production Tidak perlu Tidak perlu (build di CI/local, kirim binary)
Total dependency PHP-FPM + MariaDB + Nginx PostgreSQL + Nginx

6. Workflow Development → Production (Keseluruhan)

LAPTOP/DEV
│
├── 1. Edit kode (backend Go + frontend React)
│
├── 2. Build backend:
│      cd backend && go build -o leah ./cmd/api
│      → Hasil: file "leah"
│
├── 3. Build frontend:
│      cd frontend && npm run build
│      → Hasil: folder "dist/"
│
├── 4. Kirim ke server:
│      rsync -avz backend/leah user@server:/var/www/leah.lan/backend/
│      rsync -avz frontend/dist/ user@server:/var/www/leah.lan/frontend/dist/
│
└── 5. Restart di server:
       ssh user@server "sudo systemctl restart leah"

─── ATAU (lebih modern) ───

CI/CD (GitHub Actions)
│
├── 1. Push ke GitHub
├── 2. GitHub Actions otomatis:
│      - Build Go binary
│      - Build React frontend
│      - Kirim ke server via SSH/SCP
│      - Restart service
└── 3. ✅ Selesai (tinggal nunggu 1-2 menit)

7. Ringkasan — Apa yang Perlu Diinstall di Fresh VM

✅ Yang WAJIB di install:

  • Nginx — web server + reverse proxy + SSL
  • PostgreSQL — database

✅ Yang CUMA untuk build (bisa dihapus setelahnya):

  • Go — compile backend jadi binary
  • Node.js + npm — build frontend React jadi file statis

❌ Yang TIDAK perlu:

  • PHP — tidak perlu sama sekali
  • Docker — opsional, kalau mau cara alternatif
  • Composer — tidak ada

Perbandingan Akhir — Perintah Install

🐘 PHP Native (biasa):
  apt install nginx php8.4 php8.4-fpm php8.4-mysql mariadb-server
  → 5 package

🐹 Go LEAH:
  apt install nginx postgresql
  → 2 package (Go & Node.js hanya untuk build, tidak untuk runtime)

8. Cek Pemahaman — Benar atau Salah?

  1. "Production server harus install Go" — SALAH. Go hanya untuk build, binary sudah mandiri.
  2. "Go binary ukurannya besar karena mengandung semua dependency" — BENAR. Tapi ini justru kelebihan: gak perlu install dependency di server.
  3. "Kalau ada bug, cukup ganti file .php seperti biasa" — SALAH. Di Go, harus compile ulang binary dan restart service.
  4. "LEAH butuh PHP-FPM" — SALAH. Go punya web server sendiri.
  5. "Nginx tetap diperlukan" — TERGANTUNG. Untuk production, iya (buat SSL, static files, security). Untuk development di laptop, gak perlu.
  6. "Node.js harus selalu jalan di server" — SALAH. Node.js cuma untuk build React. Hasil build adalah file HTML/CSS/JS statis.
