🐱 LEAH

List Everything Assets & Helpdesk — Architecture & Database Documentation

1. Project Overview

LEAH is an open-source IT Asset Management & Helpdesk Ticketing system inspired by GLPI. Built with modern stack: Go backend, React frontend, PostgreSQL database. Single binary deployment, no runtime dependencies.

LanguageGo 1.24 (backend), TypeScript/React (frontend)
DatabasePostgreSQL 16+
AuthJWT (HS256), bcrypt password hashing
DeploymentSingle binary (~20MB), Nginx reverse proxy, Cloudflare Tunnel
LicenseOpen Source

2. Technology Stack

Backend: Go 1.24, Chi Router, pgx (PostgreSQL driver), golang-jwt Frontend: React 18, TypeScript, Vite, Tailwind CSS Database: PostgreSQL 16, pgxpool connections Data Fetching: TanStack Query (React Query) Routing: React Router v6, Chi Router (Go) Auth: JWT (HS256), bcrypt, Role & Permission middleware UI Components: Custom Modal, ConfirmDialog, Toast notification

3. Architecture

3-Tier Architecture:

  Browser ──→ Nginx ──→ /api/* ──→ Go API (Chi Router) ──→ PostgreSQL
                         ──→ /* ──→ index.html (React SPA)

Backend Layer Pattern:

  Handler (HTTP request/response)
     ↓
  Service (business logic, validation)
     ↓
  Repository (SQL queries — pgx)
     ↓
  PostgreSQL

Every endpoint is protected by two middleware layers:

  • Auth middleware — validates JWT token, extracts user + permissions + org scope
  • RequirePermission middleware — checks specific permission (e.g. tickets.create)

5. Role Hierarchy

Superuser (is_superuser=true — user flag, NOT a role)
  └── Bypass ALL permission checks
  └── Can manage roles & permissions
  └── Set directly via database (no API)

Superadmin (role)
  └── ALL permissions including settings.*
  └── Can manage roles & permissions via UI
  └── CANNOT create superuser accounts

Admin (role, is_admin=true)
  └── Bypass content permissions (tickets.*, assets.*, users.*)
  └── CANNOT access settings.* or Bin (403)

Agent / Editor (role)
  └── Must have explicit permissions
  └── Can only edit within own org scope
  └── Cannot delete

User (role)
  └── Can only create tickets
  └── Can view own tickets (read.own)

7. Scope & Multi-Tenant

Every data record (user, asset, ticket) is tied to an organization_id. Organizations are hierarchical within holdings using materialized paths.

Scope Filter: When a user makes a request, the middleware:

  1. Extracts org_ids and org_paths from JWT
  2. Repository queries descendant orgs via path LIKE '/parent_id/%'
  3. Adds WHERE organization_id = ANY($accessible_org_ids) to queries

Superuser bypasses all scope filters.

10. Deployment

  # Build
  cd backend && go build -o leah ./cmd/api
  cd frontend && npm install && npm run build

  # Run (production)
  DATABASE_URL="postgres://user:pass@localhost:5432/leah" ./backend/leah

  # Systemd service
  [Service]
  ExecStart=/var/www/leah.lan/backend/leah
  Environment=DATABASE_URL=postgres://...
  Environment=JWT_SECRET=...
  Restart=always

Single binary. No runtime dependencies. Nginx serves frontend/dist/ and reverse proxies /api/* to Go binary.
